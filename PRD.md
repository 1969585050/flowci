# FlowCI 产品需求文档 (PRD) v2.0

## 1. 产品概述

### 1.1 产品名称
**FlowCI** - 轻量级 Docker 镜像构建与部署工具

### 1.2 产品定位
FlowCI 是一款面向个人开发者和小型团队的桌面端 CI/CD 工具，专注于解决 Docker 镜像的构建、推送和部署问题。它提供简洁的图形界面，让用户无需编写复杂的配置文件，即可快速完成从代码到镜像再到部署的全流程。

### 1.3 核心价值
- **简单易用**：图形界面配置，无需编写 YAML
- **通用支持**：支持所有编程语言和框架
- **灵活部署**：支持本地和远程 Docker 环境
- **快速上手**：一键安装，即刻使用

### 1.4 目标用户画像

| 角色 | 描述 | 核心痛点 |
|------|------|---------|
| **个人开发者** | 独立开发者和自由职业者 | 需要快速验证和部署项目，没有时间配置复杂的 CI/CD |
| **小型团队** | 2-5人的初创团队 | 缺乏专职运维，需要简化部署流程 |
| **全栈开发者** | 同时维护前端后端的开发者 | 需要统一管理多个不同技术栈的项目 |

## 3. 非功能性需求

### 3.1 性能需求

| 指标 | 要求 | 说明 |
|------|------|------|
| 应用启动时间 | < 3 秒 | 从点击图标到界面可操作 |
| 内存占用 | < 150MB (空闲时) | 不包含 Docker 内存 |
| 安装包大小 | < 80MB | Windows installer |
| 镜像构建速度 | 与原生 docker build 相当 ±20% | 使用 docker buildx |
| 界面响应时间 | < 100ms | 用户操作到系统响应 |
| 并发构建数 | 支持 2 个并行构建 | MVP 阶段限制 |

### 3.2 安全性需求

| 需求 | 说明 | 实现方式 |
|------|------|---------|
| **凭证加密** | 所有敏感信息加密存储 | AES-256-GCM 加密 |
| **传输安全** | SSH 隧道加密传输 | 端到端加密 |
| **凭证隔离** | 不同项目的凭证隔离 | 操作系统凭证管理器 |
| **构建隔离** | 构建过程隔离运行 | Docker 容器隔离 |
| **审计日志** | 记录所有敏感操作 | 本地日志 + 可导出 |

### 3.3 可用性需求

| 需求 | 要求 |
|------|------|
| **系统可用性** | 99.9% (月故障时间 < 45 分钟) |
| **故障恢复** | 支持断点续传和自动恢复 |
| **数据备份** | 自动备份配置文件 |
| **错误提示** | 错误信息清晰，提供解决建议 |

### 3.4 兼容性需求

| 项目 | 要求 |
|------|------|
| **操作系统** | Windows 10/11 (64-bit) |
| **Docker** | Docker Desktop 4.0+ 或 Docker Engine 20.10+ |
| **内存** | 最少 4GB RAM |
| **磁盘** | 最少 10GB 可用空间 |

### 3.5 UI/UX 需求

| 需求 | 要求 | 优先级 |
|------|------|--------|
| **主题模式** | 深色/浅色主题，跟随系统切换 | P0 (MVP) |
| **自定义主题色** | 支持自定义主色调和预设主题板 | P2 (Phase 3) |
| **布局定制** | 侧边栏宽度、组件位置可调整 | P2 (Phase 3) |
| **快捷键定制** | 支持自定义键盘快捷键 | P1 (Phase 2) |
| **通知定制** | 支持通知方式、免打扰模式 | P2 (Phase 3) |
| **多语言支持** | 中文/英文 UI | P2 (Phase 3) |
| **动画效果** | 支持界面动画平滑度调节 | P2 (Phase 3) |

### 3.6 扩展性需求

## 5. 技术架构

### 5.1 技术选型

| 层级 | 技术方案 | 说明 | 选择理由 |
|------|----------|------|---------|
| **桌面框架** | **Tauri 2.0** | 轻量级桌面框架 | 体积小(~10MB)，性能好，安全 |
| **前端框架** | **Vue 3 + TypeScript** | 现代化前端框架 | 开发效率高，类型安全 |
| **UI 组件库** | **Naive UI** | Vue 3 桌面端组件库 | 轻量、支持深色主题、TypeScript 友好 |
| **主题引擎** | **CSS Variables + Naive UI 主题系统** | 动态主题切换 | 支持自定义颜色、实时预览、无闪烁切换 |
| **后端语言** | **Go** | 高性能云原生语言 | 并发处理强，交叉编译简单，学习曲线低 |
| **Docker SDK** | **docker-sdk-go** | Go 生态成熟 SDK | 官方维护，功能完整 |
| **数据存储** | **SQLite** | 轻量级数据库 | MVP 阶段足够，支持加密 |
| **配置存储** | **TOML** | 人类可读的配置文件 | 简单、明确、无歧义 |

### 5.2 技术风险与应对

| 风险 | 影响 | 应对策略 |
|------|------|---------|
| Tauri 生态较新 | 插件不如 Electron 成熟 | 关键功能自研，预留切换方案 |
| Go Docker SDK 性能 | 需要验证大规模构建 | 使用 docker buildx 提升构建性能 |
| Windows Docker 权限 | WSL2 兼容性问题 | 提供 WSL2 和 Linux 两种开发环境建议 |

### 5.5 核心模块设计

#### 5.5.1 Builder 模块
- **职责**: 负责 Docker 镜像构建
- **关键接口**:
  - `Build(ctx, config) (*BuildResult, error)`
  - `GenerateDockerfile(lang) (string, error)`
  - `ValidateSource(path) (bool, error)`

#### 5.5.2 Deployer 模块
- **职责**: 负责应用部署
- **关键接口**:
  - `Deploy(ctx, config) (*DeployResult, error)`
  - `Rollback(ctx, id) (*DeployResult, error)`
  - `GetStatus(ctx, project) (*Status, error)`

#### 5.5.3 Pipeline 模块
- **职责**: 负责流水线编排
- **关键接口**:
  - `Execute(ctx, pipeline) (*PipelineResult, error)`
  - `Validate(pipeline) (bool, error)`
  - `GetExecutionOrder(pipeline) ([]string, error)`

#### 5.5.4 Config 模块
- **职责**: 负责配置和凭证管理
- **关键接口**:
  - `LoadConfig() (*AppConfig, error)`
  - `SaveConfig(config) error`
  - `EncryptCredential(plain) (encrypted, error)`

## 8. 竞品分析

### 8.1 竞品对比矩阵

| 特性 | FlowCI | Jenkins | GitHub Actions | GitLab CI | Woodpecker |
|------|--------|---------|----------------|-----------|------------|
| **部署模式** | 桌面端 | 服务端 | 云端/自托管 | 服务端 | 容器化 |
| **学习曲线** | 低 | 高 | 中 | 中 | 低 |
| **配置方式** | GUI/YAML | Jenkinsfile | YAML | YAML | YAML |
| **Docker 支持** | ⭐⭐⭐⭐⭐ | ⭐⭐⭐ | ⭐⭐⭐⭐ | ⭐⭐⭐⭐ | ⭐⭐⭐⭐⭐ |
| **本地构建** | ⭐⭐⭐⭐⭐ | ⭐ | ⭐ | ⭐ | ⭐⭐⭐ |
| **体积大小** | ~10MB | ~200MB | 云服务 | ~1GB | ~100MB |
| **离线支持** | ⭐⭐⭐⭐⭐ | ⭐⭐⭐⭐ | ⭐ | ⭐⭐⭐⭐ | ⭐⭐⭐⭐ |

### 8.2 差异化定位

**FlowCI 的独特价值**:
1. **真正的桌面端** - 无需搭建服务端，随时可用
2. **本地优先** - 直接使用本地 Docker，无网络延迟
3. **极简配置** - GUI 配置，无需编写 YAML
4. **隐私安全** - 代码不离开本地，凭证本地加密

## 9. 风险与应对

### 9.1 技术风险

| 风险 | 影响 | 应对策略 |
|------|------|---------|
| Go Docker SDK 功能限制 | 某些高级特性不可用 | 预备方案：使用命令行封装 |
| Windows Docker 权限问题 | WSL2 兼容性 | 提供详细的环境配置指南 |
| Tauri 2.0 稳定性 | 第三方插件缺乏 | 关键功能自研，预留 Electron 迁移方案 |

### 9.2 用户体验风险

| 风险 | 应对策略 |
|------|---------|
| 配置复杂 | 提供向导模式和模板 |
| 错误信息不清晰 | 完善错误提示和解决方案建议 |
| 学习成本 | 提供新手教程和示例项目 |
